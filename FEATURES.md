# Features

Track feature state here. Keep detailed requirements in FSD/TSD/GDD documents when those exist.

## Implemented

- CI/CD and workflow automation guidance.
- Comment and summary discipline for code, scripts, PRs, and agent completion notes.
- Git/PR preflight guidance before creating branches or PRs.
- Duplicate-work protection for already-merged or already-open PR work.
- Generic reusable examples that avoid project-specific enum/domain names.
- `.agent-guidelines-version` marker for synced pack version tracking.
- Stable repository script filename policy, with versioned names reserved for distributed artifacts and generated outputs.
- `repo-seed` sync script with stable filename for updating multiple repositories.
- Instruction strictness levels (`MUST` / `SHOULD` / `MAY`).
- Strict Git Flow branch families and PR creation/proposal rules.
- No-AI-name Git artifact rule.
- Indentation preservation guidance for existing files.
- C# control-flow convention allowing single-statement guards without braces when the statement is on the next line.
- User-facing text/output style preservation, including ASCII vs Unicode punctuation and decorative separators.
- C# spacing convention requiring a blank line after completed control blocks before independent statements.
- Agent workflow guidelines pack.
- Separated C# and script coding conventions.
- Repository documentation templates and policies.
- FSD, TSD, and GDD templates.
