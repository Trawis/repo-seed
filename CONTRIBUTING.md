# Contributing

**Document role**: Repository-only contributor guidance

**Sync behavior**: Never copied into target repositories

1. Search existing issues and pull requests before starting.
2. Read `AGENTS.md` for repository rules and validation commands.
3. Branch from current `main` using `feature/<short-kebab-description>`.
4. Keep changes focused and update tests and documentation when behavior changes.
5. Add every distributed asset under `pack/files/` to `pack/manifest.json`.
6. Open a pull request to `main` describing impact, validation, and limitations.
