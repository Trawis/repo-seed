# Security Policy

**Document role**: Repository-only security guidance

**Sync behavior**: Never copied into target repositories

## Supported Versions

Security fixes are applied to the latest released major version. Older releases may receive documentation for a safe migration but are not maintained indefinitely.

## Reporting a Vulnerability

Do not disclose a suspected vulnerability in a public issue.

Use GitHub's private vulnerability reporting for this repository when available. Otherwise, contact a repository maintainer privately through the contact information on their GitHub profile.

Include:

- affected pack and script versions;
- operating system and Python version;
- reproduction steps or a minimal proof of concept;
- expected impact;
- suggested mitigation, if known.

Reports will be acknowledged when reviewed. Public disclosure should wait until a fix or mitigation is available.
